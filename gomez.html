  • Protocolo HTTP: Es el protocolo que nos permite realizar peticiones de datos y recursos en la web. Caracteristicas clave:
    • HTTP es sencillo
    • HTTP es extensible
    • HTTP es un protocolo con sesiones, pero sin estados
    Una petición HTTP se divide en tres partes: Linea de petición, cabecera y cuerpo.
    Una respuesta HTTP también se divide en tres partes: Linea de estatus, cabecera y cuerpo.
    Los métodos más importates de HTTP son POST, GET, PUT, DELETE y HEAD.
    Fuentes: 1 2 3
  • Proceso de carga de una página web (Caso de Servidor en PHP):

    2. LLega al servidor Web.
    3. El servidor obtiene la pagina.php.
    4. Se procesa el php a html.
    5. Se manda al navegador.
    Fuente: 1
  • Protocolo TCP: Es un protocolo que garantiza que los datos serán entregados en su destino sin errores y en el mismo orden en que se transmitieron. Este protocolo distingue dos o mas aplicaciones en una maquina gracias a su sistema de puertos.
    Hay determinadas aplicaciones que necesitan tener un puerto identificado con un número que no varíe estos puertos asiganados automaticamente por el sistema son los puertos bien conocidos.
    Estructura de un segmento:
    • Puerto Origen (16 bits)
    • Puerto Destino (16 bits)
    • Número de Secuencia (32 bits)
    • Número de Reconocimiento (32 bits)
    • Longitud de Cabecera (4 bits)
    • Reservado (6 bits)
    • Bits de código (6 bits)
    • Ventana (16 bits)
    • Checksum (16 bits)
    • Opciones (longitud variable)
    Un socket tipo listener, es aquel que está escuchando mediante un puerto TCP, peticiones de clientes que desean conectarse al servidor.
    Fuentes: 1 2 3
  • Protocolo IP: Se encarga del desarrollo y transporte de paquetes de datos, aunque su recepción no está asegurada. Además, forma parte del conocido protocolo TCP/IP.
    Una dirección IP es un número que identifica de manera lógica y jerárquicamente a una interfaz de un dispositivo. El enrutamiento IP es el mecanismo por el que en una red los paquetes de información se hacen llegar desde su origen a su destino final, siguiendo un camino o ruta a través de la red.
    El datagrama de Internet es el paquete de transferencia base en la familia de protocolos de Internet. Contiene una cabecera con información para IP, y unos datos que son relevantes sólo para los protocolos de más alto nivel. Fuentes: 1 2 3

  • Metologias de desarrollo:
    • Modelo en cascada:En este modelo cada etapa representa una unidad de desarrollo con un pequeño descanso en el medio. Por lo tanto, cada siguiente etapa inicia tan pronto como la anterior haya culminado, y esos descansos son usados para confirmaciones del lado del cliente.

Lenguajes de programación:

  • Lenguajes de programación:
    • Lenguajes compilados: es un lenguaje de programación cuyas implementaciones son normalmente compiladores.
    • Lenguajes interpretados: Es el lenguaje cuyo código no necesita ser preprocesado mediante un compilador, eso significa que el ordenador es capaz de ejecutar la sucesión de instrucciones dadas por el programador sin necesidad de leer y traducir exhaustivamente todo el código.
    • Compilador: Traductores que generan código de máquina a partir del código fuente.
    • Interprete: ejecutores paso a paso del código fuente, donde no se lleva a cabo una traducción en la preejecución.
    • Transpilador: es un tipo especial de compilador que traduce de un lenguaje fuente a otro fuente también de un nivel de abstracción parecido.

Control de versiones:

  • Concepto: Es un programa que se dedica a la gestión de los diversos cambios que se realizan sobre los elementos de algún producto o una configuración del mismo.
    Fuente: 1
  • Ventajas:
    • Trazabilidad:Poder observar de donde y de quien vienen los cambios.
    • Historial de cambios:Ver todos los cambios hechos sobre el proyecto.
    • Reversión:Volver a versiones anteriores del proyecto en caso de error.
    • Trabajo en grupo:Trabajar en grupo con gran facilidad.
    • Trabajo en ramas:Trabajar sobre varias ramas distintas de un mismo proyecto.
  • Herramientas:
    • Cvs:Es una aplicación informática que implementa un sistema de control de versiones.
    • Subversion svn: Es una herramienta de control de versiones open source basada en un repositorio cuyo funcionamiento se asemeja enormemente al de un sistema de ficheros.
    • Git:es un software de control de versiones diseñado por Linus Torvalds, pensando en la eficiencia y la confiabilidad del mantenimiento de versiones de aplicaciones cuando éstas tienen un gran número de archivos de código fuente.
    • Mercurial:es un sistema de control de versiones multiplataforma, para desarrolladores de software. Está implementado principalmente haciendo uso del lenguaje de programación Python, pero incluye una implementación binaria de diff escrita en C.
  • Repositorios Git:
    • Github:Es una forja para alojar proyectos utilizando el sistema de control de versiones Git.
    • Bitbucket:Es un servicio de alojamiento basado en web, para los proyectos que utilizan el sistema de control de versiones Mercurial y Git.
    • Gitlab:Es un servicio web de control de versiones y desarrollo de software colaborativo basado en Git. Además de gestor de repositorios, el servicio ofrece también alojamiento de wikis y un sistema de seguimiento de errores, todo ello publicado bajo una Licencia de código abierto.
